# 飞书机器人集成测试指南

## 测试前准备

1. **配置飞书 Webhook URL**
   - 打开 `.env` 文件
   - 将 `FEISHU_WEBHOOK_URL` 替换为你的真实飞书机器人 Webhook URL
   - 保存文件

2. **重启开发服务器**
   ```bash
   # 如果服务正在运行，先停止（Ctrl+C）
   # 然后重新启动
   coze dev
   ```

## 测试方法

### 方法一：通过网站界面测试

1. 访问网站的「联系我」页面（`http://localhost:5000/contact`）
2. 滚动到「访客留言」部分
3. 填写姓名和留言内容
4. 点击「发布」按钮
5. 检查飞书APP是否收到消息通知

### 方法二：通过 API 直接测试

使用 curl 命令测试 API：

```bash
curl -X POST http://localhost:5000/api/contact/feishu \
  -H "Content-Type: application/json" \
  -d '{
    "name": "测试用户",
    "message": "这是一条测试消息，用于验证飞书机器人集成。"
  }'
```

预期返回：
```json
{
  "success": true,
  "message": "留言已发送！我会尽快查看并回复您。"
}
```

### 方法三：使用 Postman 或其他 API 测试工具

1. 创建 POST 请求
2. URL: `http://localhost:5000/api/contact/feishu`
3. Headers:
   - `Content-Type: application/json`
4. Body (JSON):
   ```json
   {
     "name": "测试用户",
     "message": "这是一条测试消息"
   }
   ```
5. 发送请求并查看响应

## 常见问题排查

### 问题1：点击发布按钮无响应

**可能原因：**
- JavaScript 错误
- API 路由未正确加载

**解决方法：**
1. 打开浏览器开发者工具（F12）
2. 查看 Console 标签页，检查是否有错误信息
3. 查看 Network 标签页，检查请求是否发送成功

### 问题2：返回 500 错误

**可能原因：**
- 飞书 Webhook URL 未配置
- 飞书 Webhook URL 格式错误
- 飞书机器人未正确配置

**解决方法：**
1. 检查 `.env` 文件中的 `FEISHU_WEBHOOK_URL` 是否配置
2. 检查 Webhook URL 格式是否正确
3. 查看服务器日志：`tail -n 50 /app/work/logs/bypass/app.log`

### 问题3：请求成功但飞书未收到消息

**可能原因：**
- 飞书 Webhook URL 过期
- 飞书机器人权限不足
- 飞书群聊设置问题

**解决方法：**
1. 在飞书群聊中重新生成 Webhook URL
2. 更新 `.env` 文件
3. 重启服务器
4. 检查飞书机器人是否在群聊中

### 问题4：飞书消息格式显示异常

**可能原因：**
- API 响应格式错误
- 飞书消息模板版本不兼容

**解决方法：**
1. 检查飞书开放平台文档，确认消息格式
2. 查看 API 代码中的消息构建逻辑
3. 修改消息格式以符合飞书最新规范

## 性能优化建议

1. **添加请求限流**
   - 防止恶意用户频繁提交留言
   - 可以使用 Redis 或内存缓存实现

2. **添加验证码**
   - 防止机器人自动提交
   - 可以使用 reCAPTCHA 或其他验证码服务

3. **异步处理**
   - 将飞书消息发送改为异步处理
   - 提升用户体验，避免等待飞书响应

4. **消息队列**
   - 使用消息队列（如 RabbitMQ、Redis Queue）
   - 提高消息发送的可靠性

## 安全加固

1. **输入验证**
   - 验证姓名和留言内容的长度
   - 过滤特殊字符，防止 XSS 攻击

2. **频率限制**
   - 限制每个 IP 地址的提交频率
   - 可以使用中间件实现

3. **IP 白名单（可选）**
   - 只允许特定 IP 访问 API
   - 适合内部使用场景
