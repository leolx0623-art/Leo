# 飞书机器人配置指南

本指南将帮助你配置飞书机器人，实现访客留言的实时推送通知。

## 前置要求

- 拥有飞书账号
- 具有创建机器人的管理员权限（或者联系管理员创建）

## 配置步骤

### 1. 创建飞书自定义机器人

1. **打开飞书群聊**
   - 在飞书中打开你要接收消息的群聊
   - 确保你在此群中有创建机器人的权限

2. **添加机器人**
   - 点击群聊右上角的「...」菜单
   - 选择「设置」
   - 点击「机器人」
   - 点击「添加机器人」
   - 选择「自定义机器人」
   - 点击「添加」

3. **配置机器人**
   - 输入机器人名称，例如：「网站留言机器人」
   - 选择机器人头像（可选）
   - 点击「下一步」

4. **获取 Webhook URL**
   - 系统会生成一个 Webhook URL
   - 复制这个 URL（格式：`https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxxxxxxx`）
   - 保存到项目的 `.env` 文件中

### 2. 配置环境变量

打开项目的 `.env` 文件，找到飞书机器人配置部分：

```env
# 飞书机器人 Webhook URL
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxxxxxxx
```

将你复制的 Webhook URL 替换掉上面的示例 URL。

### 3. 重启服务

修改 `.env` 文件后，需要重启开发服务器以使配置生效：

```bash
# 如果服务正在运行，先停止（Ctrl+C）
# 然后重新启动
coze dev
```

### 4. 测试配置

完成配置后，访问网站的「联系我」页面，在访客留言板中留言，验证是否能够收到飞书通知。

## 消息格式

访客留言将以卡片形式发送到飞书，包含以下信息：

- **访客姓名**：留言者填写的姓名
- **留言时间**：留言提交的时间
- **留言内容**：完整的留言文本
- **快捷操作**：提供「回复邮件」按钮

## 安全建议

1. **保护 Webhook URL**
   - 不要将 Webhook URL 提交到公开的代码仓库
   - 使用 `.env` 文件或环境变量管理
   - `.env` 文件已添加到 `.gitignore` 中

2. **限制机器人发送频率**
   - 飞书机器人有发送频率限制
   - 不要在短时间内大量发送消息
   - 考虑添加防刷机制（前端已实现）

3. **设置安全策略（可选）**
   - 在飞书机器人设置中，可以配置 IP 白名单
   - 可以设置关键词过滤
   - 可以限制只有特定成员可以@机器人

## 常见问题

### Q: 为什么收不到消息？

**A:** 检查以下几点：
1. 确认 `.env` 文件中的 `FEISHU_WEBHOOK_URL` 配置正确
2. 确认 Webhook URL 没有过期（飞书机器人 Webhook 有有效期）
3. 确认服务器已重启，环境变量已加载
4. 查看服务器日志，确认 API 调用是否成功

### Q: Webhook URL 过期了怎么办？

**A:** 飞书机器人的 Webhook URL 有有效期限制（通常为几个月）。过期后需要：
1. 在飞书群聊中重新生成 Webhook URL
2. 更新 `.env` 文件
3. 重启服务器

### Q: 如何更换接收群聊？

**A:** 如果想要更换接收消息的群聊：
1. 在新的群聊中创建自定义机器人
2. 获取新的 Webhook URL
3. 更新 `.env` 文件
4. 重启服务器

### Q: 能否同时发送到多个群聊？

**A:** 当前实现只支持发送到一个群聊。如果需要发送到多个群聊：
1. 创建多个飞书机器人，获取多个 Webhook URL
2. 修改 API 代码，循环发送到所有 Webhook URL
3. 在 `.env` 文件中配置多个 URL（使用逗号分隔）

## 扩展功能

### 添加邮件通知

如果你希望在发送飞书消息的同时也发送邮件通知，可以修改 API 代码，在飞书消息发送成功后调用邮件发送接口。

### 添加数据库存储

如果需要保存所有访客留言到数据库，可以：
1. 创建数据库表（如 `guestbook`）
2. 在发送飞书消息前，先保存留言到数据库
3. 提供后台管理界面查看历史留言

### 添加敏感词过滤

为了防止垃圾留言，可以在发送到飞书前添加敏感词过滤：
1. 维护一个敏感词列表
2. 在留言提交时检查是否包含敏感词
3. 如果包含，拒绝提交并提示用户
