# 飞书机器人密钥配置指南

## 问题说明

如果您遇到"飞书机器人没有及时收到消息通知"的问题，这是因为飞书更新了安全策略，要求自定义机器人必须使用签名验证。

## 解决方案

您需要获取飞书机器人的密钥（Secret），并配置到环境中。

## 获取飞书机器人密钥步骤

### 1. 打开飞书机器人设置

1. 打开飞书APP
2. 进入已添加机器人的群聊
3. 点击群聊右上角的「...」菜单
4. 选择「设置」
5. 点击「机器人」
6. 找到您的自定义机器人，点击进入

### 2. 查看机器人密钥

在机器人设置页面中，您会看到以下信息：
- **Webhook URL**：已配置 ✓
- **密钥（Secret）**：需要获取 🔑

密钥通常显示为一串随机字符，例如：`xxxxx`

**重要提示**：
- 密钥只显示一次，请务必复制保存！
- 如果忘记密钥，需要删除机器人并重新创建

### 3. 配置环境变量

打开项目的 `.env` 文件，找到飞书机器人配置部分：

```env
# 飞书机器人 Webhook URL
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/da839b28-73d4-4572-854e-ef941033ee6f

# 飞书机器人密钥（签名验证需要）
FEISHU_WEBHOOK_SECRET=your-secret-key-here  # 替换为您的真实密钥
```

将您复制的密钥替换掉 `your-secret-key-here`。

### 4. 重启服务

修改 `.env` 文件后，需要重启开发服务器：

```bash
# 如果服务正在运行，先停止（Ctrl+C）
# 然后重新启动
coze dev
```

或者如果使用后台运行：

```bash
# 停止服务
pkill -f "coze dev"

# 重新启动
coze dev > /dev/null 2>&1 &
```

## 验证配置

### 方法一：通过网站测试

1. 访问联系页面：`http://localhost:5000/contact`
2. 在访客留言部分填写姓名和留言
3. 点击「发布」按钮
4. 检查飞书APP是否收到消息通知

### 方法二：通过API测试

使用 curl 命令测试：

```bash
curl -X POST http://localhost:5000/api/contact/feishu \
  -H "Content-Type: application/json" \
  -d '{
    "name": "测试用户",
    "message": "这是一条测试消息，用于验证签名验证配置。"
  }'
```

预期返回：
```json
{
  "success": true,
  "message": "留言已发送！我会尽快查看并回复您。"
}
```

### 方法三：查看日志

查看服务器日志，确认签名验证是否成功：

```bash
tail -n 50 /app/work/logs/bypass/app.log
```

成功的日志会显示：

## 常见问题

### Q1: 找不到密钥（Secret）在哪里？

**A:** 密钥只在创建机器人时显示一次。如果您忘记了：
1. 删除当前的机器人
2. 重新创建一个新机器人
3. 这次务必复制并保存密钥！

### Q2: 密钥配置后还是收不到消息？

**A:** 请检查：
1. 确认密钥复制正确，没有多余的空格
2. 确认服务已重启，环境变量已加载
3. 查看日志，检查是否有错误信息
4. 确认飞书机器人还在群聊中

### Q3: 配置密钥后前端报错？

**A:** 密钥配置错误不会影响前端，只影响后端API。请检查：
1. `.env` 文件格式是否正确
2. 密钥是否包含特殊字符（如引号、空格）
3. 是否正确重启了服务

### Q4: 不想配置密钥可以吗？

**A:** 不可以。飞书现在强制要求签名验证，如果不配置密钥：
- 消息可能发送失败
- 飞书返回错误：`code: 19021, msg: 'sign match fail'`

## 签名验证原理

飞书自定义机器人使用 HMAC-SHA256 签名算法：

1. **获取时间戳**：当前时间的毫秒数
2. **拼接字符串**：`timestamp + "\n" + secret`
3. **计算签名**：使用 HMAC-SHA256 算法对拼接字符串进行加密
4. **Base64编码**：将加密结果进行 Base64 编码
5. **发送消息**：在消息中添加 `timestamp` 和 `sign` 字段

飞书服务器收到消息后，会：
1. 提取消息中的 `timestamp` 和 `sign`
2. 验证时间戳是否在1小时内
3. 使用相同的算法计算签名
4. 比对计算结果与接收到的签名是否一致

## 安全建议

1. **保护密钥安全**
   - 不要将密钥提交到代码仓库
   - 不要在公共场所分享密钥
   - 定期更换密钥（如发现泄露）

2. **设置IP白名单（可选）**
   - 在飞书机器人设置中，可以配置允许访问的IP地址
   - 这样即使密钥泄露，其他IP也无法使用

3. **限制机器人权限**
   - 只在必要的群聊中添加机器人
   - 定期清理不需要的机器人
