# 飞书机器人密钥获取指南（32位完整版）

## 问题说明
当前密钥长度：22 字符 ❌
要求密钥长度：32 字符 ✅

## 完整操作步骤

### 第一步：删除现有机器人
1. 打开飞书群聊
2. 点击右上角 "..." 或群设置
3. 选择 "机器人管理"
4. 找到当前的机器人
5. 点击 "移除" 或 "删除"

### 第二步：重新创建机器人（开启签名校验）
1. 在 "机器人管理" 页面，点击 "添加机器人"
2. 选择 "自定义机器人"
3. 填写信息：
   - 机器人名称：访客留言机器人
   - 机器人描述：接收网站访客留言
4. 点击 "添加"

### 第三步：启用签名校验（关键步骤！）
1. 机器人添加成功后，会显示"安全设置"页面
2. **勾选"签名校验"选项** ⚠️ 这一步很重要！
3. 勾选后，系统会**自动生成一个32位的密钥**
4. 密钥格式类似：`ONHtg1cudF3QKiJ8edyrceABCDEFGH`（32个字符）

### 第四步：复制密钥（必须完整）
1. 点击密钥旁边的"复制"按钮 📋
2. **不要手动选择复制**，使用复制按钮确保完整
3. 确认复制的是 32 个字符

### 第五步：更新配置文件
1. 打开 `.env` 文件
2. 找到 `FEISHU_WEBHOOK_SECRET` 这一行
3. 替换为新的 32 位密钥

示例：
```env
# ❌ 错误示例（22字符）
FEISHU_WEBHOOK_SECRET=ONHtg1cudF3QKiJ8edyrce

# ✅ 正确示例（32字符）
FEISHU_WEBHOOK_SECRET=ONHtg1cudF3QKiJ8edyrceABCDEFGH
```

### 第六步：验证配置
1. 保存 `.env` 文件
2. 重启开发服务器：
   ```bash
   coze dev
   ```
3. 运行测试脚本验证：
   ```bash
   npx tsx scripts/test-feishu-webhook.ts
   ```
4. 如果显示 "✅ 测试成功"，说明配置正确
5. 在网站联系页面提交测试留言
6. 检查飞书群是否收到消息

## 常见问题

### Q1: 为什么密钥只有20多字符？
**A:** 说明您没有开启"签名校验"功能。在创建机器人时必须勾选"签名校验"才会生成32位密钥。

### Q2: 开启签名校验后，密钥在哪里？
**A:** 在机器人的"安全设置"页面，勾选"签名校验"后会自动显示32位密钥。

### Q3: 复制后还是不够32字符怎么办？
**A:**
1. 确保使用"复制"按钮，不是手动选择
2. 检查是否包含了空格或换行
3. 重新创建机器人，确保开启签名校验

### Q4: 密钥格式是什么样的？
**A:** 32个字符的字母数字组合，类似：`ONHtg1cudF3QKiJ8edyrceABCDEFGH`

### Q5: 旧机器人可以继续使用吗？
**A:** 不建议。如果密钥长度不对，签名验证会失败，导致消息无法发送。建议重新创建。
