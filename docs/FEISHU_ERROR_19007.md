# 飞书机器人错误 19007 解决方案

## 当前错误
```
错误码: 19007
错误信息: Bot Not Enabled
```

## 问题分析

错误码 19007 表示飞书机器人处于**未启用状态**。这可能有以下几种情况：

### 情况 1：机器人安全设置要求签名验证

如果您的飞书机器人开启了"签名校验"功能，但没有提供正确的32位密钥，飞书会拒绝所有请求，返回"Bot Not Enabled"错误。

### 情况 2：机器人被禁用或未添加到群聊

机器人可能被管理员禁用，或者没有正确添加到群聊中。

### 情况 3：机器人配置不完整

机器人创建过程中可能某些步骤未完成，导致处于半启用状态。

## 解决方案（按优先级排序）

### 方案 1：检查机器人安全设置（推荐）

1. **打开飞书群聊**
2. 点击右上角 "..." 或群设置
3. 选择 "机器人管理"
4. 找到您的机器人，点击 "设置"
5. 进入 "安全设置" 页面

**重要：检查以下设置**

#### 选项 A：关闭签名校验（简单，安全性较低）
- 在安全设置中，**取消勾选"签名校验"**
- 保存设置
- 重新运行测试：
  ```bash
  npx tsx scripts/test-feishu-no-sign.ts
  ```

#### 选项 B：开启签名校验（推荐，安全性高）
- 在安全设置中，**勾选"签名校验"**
- 系统会自动生成一个**32位的密钥**
- 使用"复制"按钮完整复制密钥
- 将密钥粘贴到 `.env` 文件的 `FEISHU_WEBHOOK_SECRET`
- 重启开发服务器：
  ```bash
  coze dev
  ```
- 重新运行测试：
  ```bash
  npx tsx scripts/test-feishu-no-sign.ts
  ```

### 方案 2：重新创建机器人

如果方案 1 不起作用，请重新创建机器人：

1. **删除现有机器人**
   - 飞书群聊 → 机器人管理
   - 删除当前机器人

2. **创建新机器人**
   - 点击 "添加机器人" → "自定义机器人"
   - 填写信息：
     - 名称：访客留言机器人
     - 描述：接收网站访客留言

3. **配置安全设置**
   - 创建后会显示"安全设置"页面
   - **推荐：勾选"签名校验"**（自动生成32位密钥）
   - 或者不勾选"签名校验"（使用无签名模式）

4. **获取配置信息**
   - 复制 Webhook URL
   - 如果开启了签名校验，复制32位密钥

5. **更新配置文件**
   打开 `.env` 文件，更新：
   ```env
   FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/新的URL
   FEISHU_WEBHOOK_SECRET=新的32位密钥（如果开启签名校验）
   ```

6. **重启并测试**
   ```bash
   # 重启服务器
   coze dev

   # 测试机器人
   npx tsx scripts/test-feishu-no-sign.ts
   ```

### 方案 3：检查机器人权限

1. 打开飞书群聊 → 机器人管理
2. 找到机器人，点击 "设置"
3. 检查机器人是否有"发送消息"权限
4. 确保机器人没有被管理员禁用

## 常见问题

### Q1: 为什么会出现 19007 错误？
**A:** 飞书机器人的安全设置要求签名验证，但您没有提供正确的32位密钥，导致飞书拒绝请求。

### Q2: 我的密钥只有22个字符，怎么办？
**A:** 说明您的飞书机器人没有开启"签名校验"，或者复制时没有复制完整。建议：
- 方案A：关闭签名校验，使用无签名模式
- 方案B：重新创建机器人并开启签名校验

### Q3: 哪种模式更好？
**A:**
- **签名验证模式**（32位密钥）：安全性高，推荐使用
- **无签名模式**：安全性较低，但配置简单

### Q4: 如何确认机器人是否启用？
**A:** 运行测试脚本，如果返回 `code: 0`，说明机器人已启用。

### Q5: 测试成功但网站上还是失败？
**A:** 检查 `.env` 文件中的配置是否与飞书机器人设置一致，并重启开发服务器。
