# 飞书机器人完整重新创建指南

## 当前问题
- 错误码：19007 - Bot Not Enabled
- 尝试关闭签名校验后仍然失败
- 可能是机器人配置问题

## 完整重新创建步骤

### 步骤 1：删除现有机器人

1. 打开飞书群聊
2. 点击右上角 "..." → "机器人管理"
3. 找到当前的机器人
4. 点击 **"移除"** 或 **"删除"**
5. 确认删除

### 步骤 2：创建新机器人

1. 在"机器人管理"页面，点击 **"添加机器人"**
2. 选择 **"自定义机器人"**
3. 填写信息：
   - **机器人名称**：访客留言机器人
   - **机器人描述**：接收网站访客留言，提供实时通知
4. 点击 **"添加"**

### 步骤 3：配置安全设置（关键！）

创建完成后，会自动跳转到机器人设置页面。

**重要：不要勾选"签名校验"！**

1. 找到 **"安全设置"** 区域
2. **取消勾选"签名校验"** 选项（如果有）
3. 确保没有勾选任何额外的安全验证

### 步骤 4：获取 Webhook URL

在机器人设置页面：
1. 找到 **"Webhook URL"** 区域
2. 点击 **"复制"** 按钮
3. URL 格式类似：
   ```
   https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
   ```

### 步骤 5：更新配置文件

1. 打开项目根目录的 `.env` 文件
2. 找到 `FEISHU_WEBHOOK_URL` 这一行
3. 替换为新的 Webhook URL

示例：
```env
# 飞书机器人 Webhook URL
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/新的URL

# 飞书机器人密钥（签名验证需要）
# 由于我们使用无签名模式，这个配置可以保留但不会使用
FEISHU_WEBHOOK_SECRET=ONHtg1cudF3QKiJ8edyrce
```

### 步骤 6：重启开发服务器

1. 停止当前运行的开发服务器（如果正在运行）
2. 重新启动：
   ```bash
   coze dev
   ```

或者，如果服务器正在运行，`.env` 文件的修改会自动生效。

### 步骤 7：测试配置

运行测试脚本：

```bash
npx tsx scripts/test-feishu-no-sign.ts
```

预期结果：
```

📋 配置总结：
  当前模式: 无签名模式
  密钥长度: 22 字符
```

同时，检查飞书群是否收到了测试消息。

### 步骤 8：测试网站功能

1. 打开网站的联系页面
2. 填写留言表单：
   - 姓名：测试用户
   - 留言内容：这是一条测试留言
3. 点击"发送"按钮
4. 检查飞书群是否收到消息

## 如果仍然失败

### 检查 1：Webhook URL 是否正确
- 确认复制的是完整的 URL
- URL 应该以 `https://open.feishu.cn/` 开头

### 检查 2：机器人是否在群聊中
- 确认机器人已经添加到群聊
- 在群聊中应该能看到机器人头像

### 检查 3：机器人是否有发送消息权限
- 检查机器人权限设置
- 确保有"发送消息"权限

### 检查 4：网络连接
- 确认服务器可以访问飞书API
- 检查防火墙设置

### 检查 5：查看详细日志
```bash
# 查看服务端日志
tail -n 50 /app/work/logs/bypass/app.log

# 查看前端日志（浏览器控制台）
```
